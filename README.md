# 基于RFID的小区车辆管理系统

## 项目简介

本系统是一个面向小区车辆通行场景的物联网管理平台，基于 RFID 射频识别技术实现车辆自动识别与门禁控制。系统由硬件端（ESP32 + RFID + 舵机 + OLED）和软件端（Flask Web 管理后台 + MQTT 通信）两部分组成，实现了从刷卡识别到云端判断再到物理开闸的完整闭环。

## 设计背景

随着城市化进程加快，小区车辆管理面临以下挑战：

- 传统人工登记效率低、易出错
- 纸质记录难以查询和统计
- 外来车辆难以有效管控
- 月租车辆到期难以及时提醒

本系统通过 RFID 技术实现车辆自动识别，结合物联网云平台实现远程管理，有效解决上述问题。

## 功能模块

根据课设要求，系统实现以下功能模块：

| 序号 | 模块 | 功能说明 |
|:----:|------|----------|
| 1 | 系统设置 | 修改管理员密码 |
| 2 | 信息查询 | 综合查询车辆信息、车主信息和出入信息 |
| 3 | 车主信息管理 | 车主的查询、添加、修改、删除 |
| 4 | 车辆信息管理 | 车辆的查询、添加、修改、删除，支持状态和有效期管理 |
| 5 | 出入信息管理 | 出入记录的查询、添加、修改、删除，支持多条件筛选 |
| 6 | 系统首页 | 数据统计仪表盘、最近出入记录、MQTT 连接状态监控 |

## 系统特色

- **软硬件联动**：刷卡即触发完整业务流程，无需人工干预
- **云端通信**：基于 MQTT 协议，支持远程管理和实时响应
- **多状态管理**：支持正常通行、卡片挂失、卡片过期、未登记车辆四种状态
- **记录可追溯**：每次刷卡自动保存出入记录，支持多维度查询
- **现代化界面**：响应式 Web 设计，支持多端访问

## 技术栈

### 硬件端

| 组件 | 型号/规格 | 作用 |
|------|----------|------|
| 主控芯片 | ESP32-S3 | WiFi 联网、逻辑控制 |
| RFID 读卡器 | RC522 (13.56MHz) | 读取 IC 卡 UID |
| 舵机 | SG90 | 模拟道闸开关 |
| 显示屏 | 1.3寸 OLED (SH1106) | 显示通行结果 |

### 软件端

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 通信协议 | MQTT | EMQX 公共云平台 (broker.emqx.io) |
| 后端框架 | Python Flask | 轻量级 Web 框架 |
| 数据库 | SQLite | 本地文件数据库，无需安装服务 |
| 前端 | HTML + CSS + JS | 原生实现，无第三方框架依赖 |

## 安装与运行

### 软件端部署

```bash
# 1. 进入软件目录
cd software

# 2. 安装 Python 依赖
pip install flask paho-mqtt werkzeug

# 3. 启动系统
python app.py
```

启动成功后会显示：

```
 * Running on http://127.0.0.1:5000
```

浏览器访问该地址即可进入系统。

### 硬件端部署

1. 使用 Arduino IDE 打开 `hardware/duka/duka.ino`
2. 修改 WiFi 配置（SSID 和密码）
3. 选择开发板：ESP32S3 Dev Module
4. 上传程序到开发板

## 通行判断规则

| 条件 | 结果 | OLED 显示 | 舵机动作 |
|------|------|----------|----------|
| 卡片已登记且状态正常且未过期 | APPROVED | Welcome Home | 开闸 3 秒后复位 |
| 卡片已登记但状态为挂失 | DENIED | Card Revoked | 不开闸 |
| 卡片已登记但已过期 | DENIED | Card Expired | 不开闸 |
| 卡片未登记 | DENIED | Card Not Found | 不开闸 |

## 功能演示

### 登录系统

1. 访问 `http://127.0.0.1:5000`
2. 输入用户名 `admin`，密码 `123456`
3. 点击「登录系统」进入管理后台

### 车主管理

1. 点击左侧菜单「车主信息管理」
2. 点击「添加车主」填写车主信息
3. 支持按姓名、电话、住址模糊搜索
4. 点击「编辑」修改车主信息
5. 点击「删除」删除车主（需先删除关联车辆）

### 车辆管理

1. 点击左侧菜单「车辆信息管理」
2. 点击「添加车辆」绑定车牌、车主和 RFID 卡
3. 设置车辆状态（正常/挂失）和月租到期日期
4. 支持按车牌、卡号、车主搜索

### 出入记录

1. 点击左侧菜单「出入信息管理」
2. 查看所有刷卡记录（自动生成）
3. 支持按时间范围、通行结果、关键词筛选
4. 支持手动添加、修改、删除记录

### 硬件联动测试

1. 确保 ESP32 已上电并连接 WiFi
2. 确保 Python 后台已启动（MQTT 状态显示「已连接」）
3. 使用测试卡片在 RC522 上刷卡
4. 观察 OLED 显示结果和舵机动作
5. 刷新网页查看新增的出入记录

## 常见问题

### Q: MQTT 状态显示「未连接」怎么办？

A: 检查以下几点：
1. 确保电脑已联网
2. 确保 `broker.emqx.io` 可访问（可尝试 ping）
3. 重启 `app.py` 程序

### Q: 刷卡后网页没有新记录？

A: 检查以下几点：
1. 确保硬件端 WiFi 已连接
2. 确保硬件端 MQTT 已连接
3. 确保软件端 MQTT 状态为「已连接」
4. 刷新网页查看

### Q: 如何添加新的 RFID 卡片？

A: 操作步骤：
1. 先在「车主信息管理」中添加车主
2. 然后在「车辆信息管理」中添加车辆
3. 填写车牌号、选择车主、输入新卡片的 UID
4. 设置状态为「正常」，填写到期日期

### Q: 如何获取新卡片的 UID？

A: 使用 `hardware/scan/scan.ino` 程序：
1. 上传到 ESP32
2. 打开串口监视器
3. 刷卡后会显示 UID

### Q: 数据库在哪里？如何备份？

A: 数据库文件为 `software/gate_system.db`，直接复制该文件即可备份。
